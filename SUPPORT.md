# Support

## Before requesting help

1. Read [Installation](docs/INSTALLATION.md), [Usage](docs/USAGE.md) and [Troubleshooting](docs/TROUBLESHOOTING.md).
2. Confirm the complete repository was installed, not only `SKILL.md`.
3. Record the AI client, operating system, installation path and repository commit.
4. Remove all secrets, credentials and customer data from logs and screenshots.

## Good support request

Include:

- what you attempted;
- expected and actual behavior;
- AI client and version when known;
- operating system and shell;
- installation mode: user or project;
- exact non-sensitive error message;
- relevant file path;
- minimal safe reproduction;
- checks already attempted.

Use a GitHub issue for installation, documentation, compatibility, schema or helper-script problems. Use the private security process in `SECURITY.md` for vulnerabilities.

## Audit-result questions

When asking about a finding or verdict, provide the finding ID, redacted evidence, test status, scope limitation and application stack. Do not post live target URLs, credentials, tokens, session data or private customer information in public issues.

## Unsupported requests

Support cannot assist with unauthorized access, destructive exploitation, credential attacks, denial-of-service, evasion, data exfiltration or misuse against third-party systems.
